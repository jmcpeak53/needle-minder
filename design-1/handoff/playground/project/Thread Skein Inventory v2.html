Why these work

Instrument Serif carries brand warmth; DM Sans handles density. Color codes always in mono so they never get mistaken for labels.
The low-stock strip is the one amber note on home — it's the only attention-worthy state by default.
Swatch grid uses inset shadow & overlay thread hatching so each cell reads as fiber, not a flat color tile.
Low-quantity badge inverts color (terracotta → cream) so scanning the grid surfaces what to restock in one glance.
Bottom nav adds a dedicated raised camera FAB — matches the "Snap + confirm" flow you chose as primary.

Supporting flows

Capture → confirm → detail. These aren't locked, but they're styled in the same mid-fi language so you can see the whole ecosystem.

Interaction rules

Tap a swatch → open detail. Long-press → quick sheet. Short press never loses context.
−1 is optimistic. Snackbar runs 5s with undo. Hitting it again extends the window rather than stacking.
The "running low" pill replaces the plain qty when it hits 1, so the grid visually reports scarcity.
Quantity steppers always pair with a read-only number next to the control, so the current value stays legible even when the button is under a thumb.

Design notes

Ingredients used across these screens — a starting point for a fuller system when we move to hi-fi.

Typography, color, components

Type

Skein
  • Instrument Serif — display, titles, numeric stats
Medium Seagreen
  • DM Sans — UI body, labels, buttons
DMC 959
  • JetBrains Mono — codes, timestamps, counts in table

Paper & ink

  • Neutrals do 90% of the work. Saturated color is reserved for thread data.
  • Accent (terracotta) = brand + warning; never decorative.

Thread palette

  • Actual thread color comes from the label's hex — mapped to this palette for grouping / filter chips.

Motion

  • Tap (100ms) — color press state, no movement
  • Long-press (280ms) — haptic buzz, scale 0.98, then sheet rises
  • −1 feedback — qty badge cross-fades old→new in 180ms; snackbar eases up from bottom
  • Undo window — 5s; if missed, the change writes to history

Open questions for hi-fi

Capture

  • Do we need auto-shutter when the label is locked, or require tap?
  • Handling of unreadable labels — retake or drop into manual form?

Grouping

  • Color-family groups are opinionated — expose the grouping rule as a setting?
  • Secondary sort: by code vs. by hue position inside each family.

Scale

  • With 400+ skeins and 15+ groups, do we collapse groups by default and offer "jump to family"?

Projects

  • "Projects" tab is stubbed in the nav — is that in-scope now or later?
